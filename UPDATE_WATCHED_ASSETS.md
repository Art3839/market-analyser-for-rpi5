"""
Обновление: Управление отслеживаемыми активами
================================================

Добавлены новые возможности для управления активами, по которым отправляются сигналы:

1. ГРАФИЧЕСКИЙ ИНТЕРФЕЙС (GUI)
==============================

📌 Выбор активов:
  - Верхняя панель содержит выпадающее меню "Select: [Актив]"
  - Нажмите "📋 Manage" для открытия менеджера активов
  - В менеджере выберите активы кнопками с галочками
  - Нажмите "✅ Save" для сохранения

📌 Анализ:
  - При нажатии "📊 Analyze" система анализирует ТОЛЬКО выбранные активы
  - Если выбран конкретный актив в выпадающем меню - анализируются только он
  - В противном случае анализируются все отслеживаемые активы

📌 Сохранение:
  - Выбранные активы сохраняются в data/watched_assets.json
  - При перезагрузке выбор восстанавливается


2. TELEGRAM БОТ
===============

📌 Команды:
  /start    - Главное меню
  /help     - Справка
  /status   - Показать текущий статус
  /manage   - Управлять активами

📌 Управление активами:
  - Нажмите кнопку "⚙️ Управлять активами"
  - Выбирайте активы нажимая на кнопки (✓/✗)
  - Нажмите "✅ Готово" для сохранения
  - Сигналы будут отправляться только для выбранных активов

📌 Статус:
  - Команда /status показывает список отслеживаемых активов
  - Отображает количество активов в отслеживании


3. ОСНОВНАЯ ПРОГРАММА
=====================

📌 Запуск:
  - При запуске программа загружает список отслеживаемых активов из watched_assets.json
  - Анализирует только отслеживаемые активы
  - Отправляет сигналы только для них

📌 Динамическое обновление:
  - При каждом цикле программа перезагружает список отслеживаемых активов
  - Это позволяет менять активы во время работы программы через GUI или Telegram


4. ФАЙЛ РАБОТЫ
==============

data/watched_assets.json - Сохраняет список отслеживаемых активов:

{
  "watched": [
    "AAPL",
    "MSFT",
    "BTC-USD",
    "ETH-USD"
  ]
}


5. ПРИМЕРЫ ИСПОЛЬЗОВАНИЯ
==========================

📌 В GUI:
  1. Запустите: python main.py --gui
  2. Нажмите "📋 Manage"
  3. Выберите нужные активы
  4. Сохраните выбор
  5. Нажмите "📊 Analyze" для просмотра сигналов

📌 В Telegram:
  1. Напишите боту: /manage
  2. Выберите активы кнопками
  3. Нажмите "✅ Готово"
  4. Теперь будут отправляться только сигналы выбранных активов

📌 Программно (добавить актив):

from config import DATA_DIR
from utils import save_json

watched = {
    'watched': ['AAPL', 'MSFT', 'GOOGL', 'BTC-USD', 'ETH-USD']
}
save_json(watched, 'watched_assets.json')


6. СТРУКТУРА ДАННЫХ
====================

Telegram Notifier отслеживает активы:
  - notifier.watched_assets - список отслеживаемых активов
  - notifier.is_asset_watched(symbol) - проверить актив
  - notifier.add_watched_asset(symbol) - добавить актив
  - notifier.remove_watched_asset(symbol) - удалить актив
  - notifier.toggle_watched_asset(symbol) - переключить

GUI отслеживает активы:
  - self.watched_assets - список активов
  - self.watched_combo - выпадающее меню
  - self._show_asset_manager() - менеджер активов


7. ИНТЕГРАЦИЯ С СИСТЕМОЙ
=========================

✓ Telegram: Автоматически проверяет, отслеживается ли актив перед отправкой
✓ GUI: Показывает только сигналы отслеживаемых активов в таблице
✓ Main.py: Анализирует только отслеживаемые активы в цикле
✓ Все компоненты синхронизированы через shared JSON файл


8. ОБРАТНАЯ СОВМЕСТИМОСТЬ
===========================

Если watched_assets.json не существует:
  - Используется значение по умолчанию: ['AAPL', 'BTC-USD']
  - Файл создается при первом сохранении
  - По умолчанию - минималистичный набор


9. ВОЗМОЖНЫЕ ПРОБЛЕМЫ И РЕШЕНИЯ
================================

❓ Сигналы не приходят:
  ✓ Проверьте, что актив добавлен в watched_assets
  ✓ Проверьте логи в logs/market_analyzer.log

❓ GUI не обновляет список:
  ✓ Переключитесь на другую вкладку и обратно
  ✓ Перезагрузите программу

❓ Telegram бот не отвечает:
  ✓ Проверьте токен и chat_id в config.py
  ✓ Убедитесь что Telegram не блокирует вашего бота
